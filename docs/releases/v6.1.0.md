# Release Notes — Transkryptor v6.1.0

Date de préparation : 2026-06-25
Branche cible : `main`

## Résumé

Transkryptor v6.1.0 livre une refonte visuelle complète de l'interface web sous l'identité **lesur.ai**, tout en préservant strictement le layout existant, l'ensemble des IDs/classes utilisés par le JavaScript, et l'intégration backend Cloud Temple LLMaaS.

Aucun changement fonctionnel côté pipeline transcription / analyse / synthèse / diarisation : les contrats API sont identiques à la v6.0.0.

## Highlights

- Nouvelle identité visuelle **lesur.ai** : canvas cream, sidebar/header/logs en navy, accents cyan + amber, font serif Newsreader pour les titres.
- Marque T en SVG inline dans le header (5 nœuds), fidèle au mockup design source.
- Crédit "Built with ❤️ by lesur.ai" sous le panneau *Prompts avancés*, avec lien externe.
- Couleurs du chart de performance pilotées par CSS custom properties — le chart suit automatiquement le design actif.
- READMEs refondus : `README.md` (anglais, nouveau défaut) + `README_FR.md`, avec switch de langue en haut.
- Layout, IDs, classes, et selectors JS strictement préservés.
- Aucune nouvelle dépendance npm.

## Notes Techniques

### Rendu serveur

- `index.html` est désormais lu une fois au démarrage et stocké en mémoire comme template.
- À chaque requête HTTP qui sert l'application, le serveur retourne le template après une substitution textuelle minimale.
- `express.static` est configuré avec `{ index: false }` pour que la route catch-all serve toujours le template rendu.

### Chart.js

- Les couleurs (`borderColor`, `backgroundColor`, `pointBackgroundColor`, axes, grilles) sont lues via `getComputedStyle(document.documentElement).getPropertyValue('--chart-line')` etc. au moment de l'init.
- Tokens disponibles : `--chart-line`, `--chart-fill`, `--chart-axis`, `--chart-grid-y`, `--chart-grid-x`.

### i18n

- Le titre `<title>` n'est plus lié à `data-i18n="page.title"` ; il est posé programmatiquement par `i18n.js`.
- `i18n.js` choisit le titre approprié en fonction du contexte de design actif (clé `page.titleCloudTemple` disponible pour le contexte historique Cloud Temple).

### Marque T

- SVG inline, viewBox `-4 -4 44 38`, 5 cercles + 2 traits en T.
- Position et couleurs alignées sur le mockup source : 2 cercles cyan en haut-gauche/droite, 1 cercle amber large au centre haut, 1 cercle amber petit au milieu, 1 cercle blanc en bas (inversion contrastée du dark navy original).

## Accessibilité

- Contraste AA restauré sur le texte d'explication de diarisation dans la sidebar dark.
- Contraste AA restauré pour les inline `code` dans le tab content.
- Mobile (<768px) :
  - Sidebar repliée à 80px (icônes uniquement).
  - Toggle "Détecter les participants" reste visible (regression fixée).
  - Icônes des boutons d'action préservées via masquage ciblé du label texte (regression fixée).

## Compatibilité

- Backend, endpoints API et contrats JSON inchangés depuis v6.0.0.
- Les variables d'environnement requises restent `CLOUD_TEMPLE_API_KEY` et `CLOUD_TEMPLE_ALLOWED_MODELS`. `PORT` reste optionnel.
- Aucune migration de données ni de configuration nécessaire.

## Limites Connues

- Le chart de performance est initialisé une seule fois ; un changement dynamique de design en cours de session ne re-colore pas le chart (rafraîchir la page suffit, mais ce n'est pas un scénario utilisateur attendu).
- Chart.js et Marked restent chargés via CDN.
