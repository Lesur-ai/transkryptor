# Release Notes — Transkryptor v6.0.0

Date de préparation : 2026-06-25
Commit cible : `79453f1`
Branche cible : `main`

## Résumé

Transkryptor v6.0.0 transforme la base Cloud Temple-only v5 en une expérience multilingue plus complète : interface FR/EN, transcription avec hint de langue, synthèse dans une langue cible, prompts configurables et détection optionnelle des participants par LLM.

Cette release reste fidèle au périmètre SecNumCloud : aucun fournisseur OpenAI/Anthropic n'est réintroduit, aucune dépendance de diarisation externe n'est ajoutée, et la clé Cloud Temple reste côté serveur.

## Highlights

- UI bilingue français / anglais.
- Transcription multilingue avec auto-détection ou hint explicite.
- Synthèse multilingue, indépendante de la langue audio.
- Cinq presets de synthèse et un mode prompt personnalisé.
- Diarisation textuelle optionnelle par Cloud Temple LLMaaS.
- Streaming SSE des tours de parole pendant la diarisation.
- Onglet Locuteurs avec timestamps approximatifs, renommage local et export.
- Allowlist stricte des modèles conservée côté serveur.

## Changements Fonctionnels

### Interface et i18n

- Ajout d'un module i18n vanilla sans dépendance.
- Ajout des fichiers `src/client/i18n/fr.json` et `src/client/i18n/en.json`.
- Persistance de la langue UI dans `localStorage`.
- Support de `?lang=fr` et `?lang=en`.

### Langues Audio et Synthèse

- Ajout du sélecteur de langue audio.
- Transmission du hint de langue à Whisper Cloud Temple.
- Ajout du sélecteur de langue de synthèse.
- Badge de langue détectée quand disponible.

### Prompts de Synthèse

- Ajout des presets `executive`, `meeting`, `actions`, `verbatim`, `thematic`.
- Ajout du mode `custom`.
- Validation serveur : prompt personnalisé limité à 8000 caractères.
- Le prompt personnalisé reste compatible avec la langue cible grâce à une instruction serveur.

### Diarisation

- Ajout de `/api/diarize`.
- Diarisation basée sur le texte et les segments Whisper, pas sur la voix.
- Streaming des tours via SSE.
- Filtrage des IDs de segments invalides ou hallucinés.
- Réconciliation si la couverture initiale est inférieure à 90%.
- Échec si la couverture finale est inférieure à 50%.
- Avertissement utilisateur si la couverture est partielle entre 50% et 99%.

## Notes Techniques

- `audioProcessor.processAndTranscribeInChunks` retourne maintenant `{ text, segments }`.
- `appState.results` stocke `whisperSegments`, `diarization` et `speakerNames`.
- Les budgets de tokens restent à 16384 pour l'analyse et 8192 pour la synthèse.
- Les modèles reasoning Cloud Temple sont appelés avec `enable_thinking:false`.
- Aucune nouvelle dépendance npm applicative n'est nécessaire pour les fonctionnalités v6.0.

## Compatibilité

- Le pipeline sans diarisation reste compatible avec la v5.4.
- La configuration `.env` v5.1 reste valide si `CLOUD_TEMPLE_ALLOWED_MODELS` et `CLOUD_TEMPLE_API_KEY` sont définis.
- Les clés OpenAI/Anthropic ne sont plus utilisées depuis le pivot v5.

## Limites Connues

- La diarisation est une inférence textuelle par LLM ; elle ne remplace pas une diarisation audio spécialisée.
- Si Whisper ne retourne pas de segments, l'onglet Locuteurs perd les timestamps exploitables.
- Les presets de synthèse sont rédigés en français ; la langue cible est imposée par instruction.
- Chart.js et Marked restent chargés via CDN dans l'interface web.
- Validation E2E complète avec vraie clé Cloud Temple et audio multi-locuteurs recommandée avant publication publique.
